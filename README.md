# 사용한도 성분 퀴즈 — 맞춤형화장품 조제관리사

화장품 안전기준 등에 관한 규정 별표2(보존제·자외선차단제·염모제·기타 성분 사용한도)를
암기용 사지선다 퀴즈로 만든 웹앱입니다. **총 258문항**.

## 파일 구성
- `build.py` — **원본 소스.** 성분 데이터를 여기서 관리하고, 실행하면 `index.html`이 생성됩니다.
- `index.html` — 빌드된 퀴즈 앱 (이 파일 하나만 있으면 브라우저에서 바로 실행)
- `reference.md` — 원본 표 전체 참고 문서

## 퀴즈 특징
- **보기 자동 셔플** — 문제를 열 때마다 A/B/C/D 순서가 무작위로 바뀝니다.
- **헷갈리는 오답** — 오답 보기는 같은 카테고리에서 수치가 가장 가까운 다른 성분의 실제 한도, 또는 같은 성분의 다른 조건 수치입니다.
- **문제 유형 4종** — 한도(165) / 조건부(71) / 부정형(5) / 복합(17)
- **즉시 정답 공개 + 비교표** — 답을 고르면 바로 정답/오답 + 유사 항목 수치 비교표 표시.
- **출제 문항 수 선택** — 10 / 20 / 30 / 전체
- **틀린 문제만 다시 풀기** — 결과 화면에서 방금 틀린 문제만 재도전.
- **누적 오답 복습 (자동 저장)** — 여러 판에 걸쳐 틀린 문제가 브라우저에 자동 저장돼, 언제든 "누적 오답"만 모아서 풀 수 있어요. 맞히면 오답 은행에서 서서히 빠집니다.
- **OX 퀴즈 모드** — 상단 토글로 전환. 성분 데이터에서 O/X 문장을 자동 생성합니다(참: 실제 수치 / 거짓: 헷갈리는 인접 수치). 답을 고르면 정답과 해설을 보고 넘어가는 차분한 방식이에요. OX 모드는 **기록 없이 가볍게** 동작해서 누적 저장·약점 분석에는 반영되지 않습니다.
- **약점 분석** — 카테고리별 누적 정답률을 막대로 표시. 한 판이 아니라 **누적 기록** 기반이라 신뢰도가 높고, 응시 수(분모)를 함께 보여줍니다. 10문제 미만인 분류는 `·표본↓`로 표시해 섣부른 판단을 막아요.

## 데이터 저장 방식
누적 통계와 오답 기록은 **브라우저 localStorage**에 저장됩니다.
- 같은 브라우저·같은 기기에서만 유지됩니다. (기기/브라우저가 바뀌면 기록은 공유되지 않아요.)
- GitHub Pages 등 실제 웹에 올리면 정상 작동합니다.
- 약점 분석 패널의 "기록 초기화" 버튼으로 언제든 리셋할 수 있어요.

## 실행 방법
`index.html`을 브라우저로 열면 끝입니다. 설치·서버 불필요.

## GitHub Pages 배포
1. 저장소에 `index.html` 업로드 (`build.py`, `reference.md`도 함께 두면 좋습니다)
2. **Settings → Pages → Source: Deploy from a branch → main / (root) → Save**
3. `https://[아이디].github.io/[저장소이름]/` 로 접속

## 문제 수정·추가 방법
`index.html`을 직접 고치지 말고 **`build.py`를 고친 뒤 다시 빌드**하세요.

```bash
python3 build.py    # index.html이 새로 생성됩니다
```

- **단일 한도 성분** → `SIMPLE` 리스트에 `("카테고리", "성분명", "한도", "비고")` 추가
- **조건별 한도 성분** → `CONDITIONAL` 리스트에 추가 (조건마다 문제 1개씩 자동 생성)
- **부정형·복합형 문제** → `HANDWRITTEN` 리스트에 직접 작성

문제 ID는 문제 내용 기반으로 자동 생성되므로, 기존 문제를 건드리지 않으면 누적 통계도 그대로 이어집니다.
